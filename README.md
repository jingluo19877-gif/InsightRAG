## 项目背景
- 旨在解决医学影像分析中医生工作繁重、经验依赖性强的问题，提供智能辅助工具，提高诊断效率；
- 辅助放射科医生进行X光片等医学影像的初步筛查；
- 融合了多模态大模型（BiomedCLIP + MedGemma）、支持ROI区域精准检索、可解释性热力图、加权融合检索。


## 核心架构
![系统架构图](doc/images/structure2.png)

关键模块及其作用：
- 模块A：前端交互界面（Streamlit）
负责用户上传医学影像、圈选感兴趣区域（ROI）、输入主诉，并动态展示检索到的相似病例、热力图叠加效果及最终的中英文诊断报告。同时管理多会话对话历史，支持会话的新建、重命名和删除。
- 模块B：多模态特征提取与检索引擎（BiomedCLIP + ChromaDB）
使用BiomedCLIP模型将用户影像（全局或ROI）编码为特征向量，并在ChromaDB向量库中检索最相似的历史病例。支持加权融合检索（结合全局和ROI特征），返回相似图像及对应的原始报告。
- 模块C：可解释性分析模块（热力图生成）
通过计算查询图像与历史病例图像的区域特征相似度，生成空间热力图并叠加在病例图像上，直观揭示在判断相似性时关注的病灶区域，增强诊断依据的可信度。
- 模块D：大语言模型生成模块（MedGemma + Qwen）
将用户主诉通过Qwen模型翻译为英文，与检索到的病例上下文拼接后，调用MedGemma生成结构化的英文诊断报告（包含影像学表现和诊断意见），再经Qwen翻译为中文，最终以双语形式呈现。
- 模块E：会话与数据管理模块
负责多会话对话历史的持久化存储（JSON文件），记录每次交互的输入图像、检索结果、热力图路径及生成的报告，确保用户可在不同会话间切换并追溯历史诊断过程。


## ✨ 主要功能
- **功能1**：影像上传与ROI交互：支持上传X光影像（PNG/JPG/DICOM），用户可在影像上自由圈选病灶区域（ROI），系统自动保存该区域用于精准检索，实现局部特征增强的相似病例匹配。
- **功能2**：多模态相似病例检索：基于BiomedCLIP提取全局或ROI图像特征，在ChromaDB向量库中检索最相似的历史病例，支持加权融合检索（全局+ROI）以平衡整体与局部相关性，返回最匹配的影像及对应报告。
- **功能3**：可解释性热力图：针对每个检索到的相似病例，通过特征相似度映射生成跨图热力图，叠加显示在病例图像上，直观展示匹配区域，增强诊断依据的可信度和可解释性。

## 实验结果与分析
- 在 IU X-Ray 数据集的 1200 张测试影像上评估了不同检索策略的性能。全局特征检索（基于 BiomedCLIP）在 top-5 召回中的平均余弦距离为 0.071（距离越小表示越相似），表明模型能够有效对齐全局影像特征。检索结果高度相关。
- 引入 ROI 加权融合检索（α=0.5，即全局与 ROI 特征等权）后，平均距离略微下降至 0.068，但改善幅度有限。
- 在针对特定疾病的测试中，发现模型存在明显局限性。例如，当输入一张明确表现为肺实变的肺炎 X 光片时，系统生成的报告未能识别肺炎征象。

## 快速开始
### 数据集链接
https://www.kaggle.com/datasets/raddar/chest-xrays-indiana-university
### 模型服务配置
BiomedCLIP：用于图像特征提取，运行在本地。open-clip-torch 会自动从 HuggingFace Hub 下载模型权重，无需额外配置。
```bash
curl -fsSL https://ollama.com/install.sh | sh
ollama pull dcarrascosa/medgemma-1.5-4b-it:Q8_0
ollama pull qwen2.5:1.5b
# 启动Ollama服务（默认监听11434端口）
ollama serve
# 在AutoDL实例上安装ngrok并启动隧道：
ngrok http 11434
```
### 环境配置
```bash
git clone https://github.com/jingluo19877-gif/Multimodal-RAG-Medical-Test.git
cd Multimodal-RAG-Medical-Test
pip install -r requirements.txt
```
### 运行系统
```bash
streamlit run app1.py
```
